函数 -- Function 1.定义 函数也是一个变量,函数名就是函数变量的名字,函数也叫方法,也是执行序列 1.1 声明定义 function 函数名(参数列表){执行序列} 1.2 函数变量 var 函数名 = function(参数列表){执行序列} 1.3 new创建函数 var 函数名 = new Function(参数列表,执行序列) 2.变量作用域 全局变量:在script块中定义的变量 局部变量:在花括号内部声明的变量,当花括号结束,局部变量生命周期也结束了(for和if除外) 3.函数参数 js在函数调用时,对参数不做任何限制,但是在函数内部,有一个系统变量:arguments,用来管理所有传入的参数 arguments的使用,是类数组对象(部分功能和数组一样), 可以使用小标访问元素,可以用length表示长度,可以用for遍历 但是不能使用数组的api 作用:检验参数个数,模拟函数重载等 4.匿名函数,匿名函数又称为立即执行函数,创建一个函数后,立刻调用,调用完之后,立即释放,解决内存. 匿名函数不允许直接定义,必须要在定义的时候执行或者赋值给另一个函数变量, 也可以作为函数的返回值 5.回调函数 将一个函数A作为参数传递给另一个函数B,由一个函数B来决定函数A在什么时候调用 百度百科: (回调函数就是一个通过 函数指针 调用的函数. 如果你把函数的指针(地址)作为 参数传递 给另一个函数,当这个指针被用来调用其所指向的函数时,我们就说这是回调函数. 回调函数不是由该函数的实现方直接调用,而是在特定的事件或者条件发生时,由另外的一方调用的,用于对该事件或者条件进行响应.) 作用: 先绑定两个函数(操作)之间的联系, 至于什么时候回调 参数传递过来的函数A(回调函数), 是由函数B(触发函数)决定, 界面控件绑定的事件函数,就是回调函数 触发函数决定回调函数调用的时机 类似机制: 定义一个回调函数 提供函数实现的一方在初始化的时候,将回调函数的函数指针注册给调用者 当特定的事件或者条件发生的时候,调用者使用函数指针调用回电函数对事件进行处理 eg: sort(函数变量) 问题:回调函数与普通函数内调函数的区别? 满足特定事件或条件的情况下, 回调函数,可以通过参数的方式,被调用者可以随时更改, 普通函数内调用函数,如果存在多个需求,多个目的,那么调用者要不断更改代码,或者添加逻辑判断. // 回调函数的定义和使用 // 触发函数(满足特定事件或者条件的调用者) function getCall(callfn,data){ console.log("触发函数"); callfn(data); } // 回调函数(被调用者) function callBack(data){ console.log("回调函数") console.log("数据:",data); } getCall(callBack,100); 6.闭包 closure 闭包解决的问题: 全局变量随处可用,易被污染 局部变量不被污染,但不可以重用 闭包提出了一个既可以重用,又包含局部变量的机制, 闭包就是能够读取其他函数内部变量的函数 作用:提供一个可以重用,又受局部变量保护的变量,以函数的形式 eg: function test(num){ var i = num; return function(){ return i; } } 代码定义: eg: function outer(){ var n =1 ;//受保护的局部变量 function inner(){ //操作局部变量的内层函数 return n++; } return inner; //因为内部函数inner使用n这个变量,所以在outer函数调用结束的时候,n变量的声明周期并没有结束 } var opr = outer(); //n在outer()中声明出来 //outer()只调用了一次,n就声明了一次,初始化了一次,所以n就一份 console.log(opr()); //1 console.log(opr()); //2 // 此时的n只有一份 var opr2 = outer(); //再声明了一个n //此时n就有两份,上面一份跟下面一份是两个不同的n console.log(opr()); //3 console.log(opr2()); //1 解释: 变量n受局部变量机制保护,避免全局变量污染 变量n可以在全局通过函数进行获取. 7.异常处理 ERROR 什么是异常:会导致程序无法继续运行,其后的执行序列都无法正常执行 7.2 捕获异常 在调用外部接口(程序会遇到一些不可控的错误)的,不希望中断程序,而是把错误捕捉到 在错误发生后,希望对错误做统一处理,可以捕获异常
