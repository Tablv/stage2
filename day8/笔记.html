计算机网络 1.计算机硬件 cpu (计算,处理数据); 内存 (临时加载数据--频繁使用的数据,在断电或者硬件故障的时候数据消失); 硬盘 (持久化保存数据,读取速度慢); 网卡 (交换数据) 2网络 2.1 网络协议 --所有在网络中传输的数据都是带格式的 对于web应用来说,使用的网络协议是 http协议(http协议是tcp协议的一种) 一个应用程序,如果需要使用网络,那么他需要向操作系统发起申请, 申请成功后,会得到一个端口数据,用来做在本机上的唯一标识 2.2 tcp协议的特点:稳定,纠错,续传 tcp的三次握手协议: 百度补全 2.3 HTTP协议 是tcp协议的一种,是在tcp协议上的一个封装 hyperText Transfer Protocol 超文本传输协议 1> http协议是互联网应用, 客户端与服务端之间进行数据通信的一种协议,规定了请求与相应的格式 2> http采用了请求到响应的工作方式 请求与响应都需要按照http协议的格式来组织数据,这个工作是由浏览器自己完成 如果想修改http中相关参数,js不可以,python的网络爬虫可以 2.4 http特点 1)简单快捷 2)灵活 ---http兼容了几乎任意格式的数据对象 3)无连接 每次请求得到相应后,即断开连接(像音频 视频直播,肯定不能采用这种方式) 4)无状态 数据在传输过程中,不保存任何历史信息和状态信息 5)传输可靠性高 采用tcp作为传输层 6)应用性广 (浏览器与http类似强绑定) 2.5 http报文格式 http请求头 1) accept:指定客户端能够接受的内容类型 2) aceept-encoding : 浏览器支持的解码压缩格式 3) aceept-language : 浏览器语言环境 4) connection : 当前连接状态 值,keep-alive:表示保持连接 5) host : 请求服务器的地址 6) Referer : 表示当前请求来自于哪里 7) user-agent : 浏览器信息 8) Cookie : 服务端在客户端存储的少量数据 9) Content-Type : 请求体的数据类型 10) Content-Length : 消息正文的长度 http响应头 1> Date:响应返回的时间 2) Content-Length: 响应消息正文的长度 3) Content-Type : 响应消息的数据类型 text/html 文本格式 4) Server : 服务端的信息 5) Cache-Control : 服务端告诉客户端如何控制缓存 6) set-Cookie : 设置Cookie 7) Etag : 给当前资源的标识,如果服务端资源发生变化,这个Etag也会发生相应变化 2.6 url 统一资源定位符,是互联网上用来标识某一处资源的地址 组成: 协议+域名(ip)+端口(应用程序在计算机中的标识)+目录结构+参数部分+锚点名 eg: http://www.baidu.com:80/news/20190126/index.html?key=value#aname web服务器也是应用程序,默认占用的系统端口是80 2.7 状态码 标识一个http响应的状态,用一个三位数字来表示 1) 1打头 指示信息,表示请求已经接受,继续处理 2) 2打头 表示请求成功 200是一般正常访问的状态码 3) 3打头 表示重定向(需要客户端进一步操作,才能完成请求) 302 在其他地址发现了请求数据 304 表示服务端资源没有发生变化(说明客服端缓存资源是最新的,实际上没有访问服务端) 4) 4打头 客户端错误,妨碍了服务器处理 400 请求语法错误 403 拒绝访问 404 未找到资源 405 请求类型不允许 5) 5打头 服务器错误或者异常 500 服务器内部错误 502 路径错误 504 请求超时 505 http版本不支持 2.8 服务端的搭建 应用程序是一个可以直接向操作系统申请资源的执行序列, js的运行时在浏览器中运行的,不可以直接跟操作系统交互 所以服务端的应用程序是不可以用js写的,而用其他语言(c,Java,Python,php) python+Django搭建服务端 python跟js不同,是独立运行的语言 python的代码是由python的解释器去解释执行,这个解释器就是python.exe python的代码都可以打包成一个独立运行的应用程序 Django是python的web框架,提供方便操作的命令和接口, 用来搭建服务端,也是目前最主流的web服务框架之一. Django的安装,命令安装 pip install Django 搭建Django的服务端命令: 1. 创建Django工程 django-admin startproject 项目名 2. 启动Django工程 python manage.py runserver ip:端口 Django工程文件说明: urls.py 配置服务端的url地址,一个url就是一个目录(概念参照http协议);
